## p1 All-in-one Gun（Codeforces Round 1081 C(Div. 2)）（前后缀加贪心,一道用空间换时间的典型题,将区间问题转化为单点问题）

> [!note]
>
> >***题目描述***
> >
> >您正在开发一款新的射击游戏，但由于市面上有很多射击游戏，因此您决定在游戏中加入一些独特的元素。
> >
> >您有一把多功能枪，可以按照固定顺序发射子弹。弹夹中有 $$n$$$ 发子弹，其中 $$$i$$$ 发子弹会造成 $$$a_i$$$ 伤害。敌人的生命值从 $$$h$$$ 开始，当生命值变成 $$$\le 0$$时敌人就会死亡。
> >
> >该枪每秒发射一颗子弹。发射完所有 $$n$$$ 子弹后，必须重新装弹，这需要 $$$k$$秒。装弹时，子弹顺序总是相同的。不能提前装弹，必须先清空弹夹。开始时，弹夹已经装满。
> >
> >战斗开始前，您最多可以进行一次***交换：选择任意指数 $$1 \le i \lt j \le n$$$ ，用 $$a_j$ 交换 $$a_i$$ 。
> >
> >您的任务是找出杀死敌人所需的最少秒数，同时考虑到这次可选的单次交换。
>
> >***思路*** 这题显然是道贪心，我们考虑交换一对数据，使得需要的时间尽量短。在此题中，我们发现如果去暴力，复杂度为$O(n^2)$,会$TLE$。想到和连续和有关，我们会考虑前缀和的思路。
> >
> >**难点一** 前缀和怎么更新
> >
> >我一开始想差分,发现不行。那换个思路，我们不在原数组更新，我们添加新的数组，使得可以用$O(1)$的复杂度计算此时进行交换后的前缀和。贪心发现，在i位置时，交换前方的最小值和后方的最大值如果能变大，则为最大前缀和。那我们同时用minn来记录从前往后区间最小值，maxn记录从后往前区间最大值。那每次交换后新的前缀和为prefix[i]-minn[i]+maxn[i+1].这题能这么做的根本是每个状态时独立的，前缀和的更新是不可传递的，所以直接利用数学转化来计算最小值点和最大值点的交换对数值得影响即可
>
> >code
> >
> >```c++
> >#include <bits/stdc++.h>
> >using namespace std;
> >#define int long long
> >#define endl "\n"
> >void solve()
> >{
> >    int n, h, k;
> >    cin >> n >> h >> k;
> >    vector<int> arr(n + 1);  
> >    vector<int> minn(n + 1); 
> >    vector<int> pref(n + 1, 0); 
> >    int sum = 0;
> >    int minx = 1e9 + 5;
> >    for (int i = 1; i <= n; i++)
> >    {
> >        cin >> arr[i];
> >        sum += arr[i];
> >        pref[i] = pref[i - 1] + arr[i];
> >        minx = min(arr[i], minx);
> >        minn[i] = minx;
> >    }
> >    vector<int> maxx(n + 2, -1e18);
> >    for (int i = n; i >= 1; i--)
> >    {
> >        maxx[i] = max(maxx[i + 1], arr[i]);
> >    }
> >    int r = (h - 1) / sum;      
> >    int hh = h - r * sum;       
> >    int minnn = n; 
> >    for (int i = 1; i <= n; i++)
> >    {
> >        if (pref[i] >= hh) 
> >        {
> >            minnn = i;
> >            break;
> >        }
> >        int need = hh - pref[i];
> >        int maxb = maxx[i + 1]; 
> >        int minf = minn[i];      
> >        if (maxb - minf >= need) 
> >        {
> >            minnn = i;
> >            break;
> >        }
> >    }
> >    int ans = r * (n + k) + minnn;
> >    cout << ans << endl;   
> >}
> >signed main()
> >{
> >    int t;
> >    cin >> t;
> >    while (t--)
> >    {
> >        solve();
> >    }
> >    return 0;
> >}
> >```

> > ***题目描述***
>
> > 小苯有一个长度为 $n$ 的序列 $a_1, a_2, \dots, a_n$，每个元素都是正整数。
> > 小苯每次操作可以选择序列中的一个元素 $a_i$，将其替换为任意正整数。
> > 小苯想知道，最少需要修改多少个元素（可以不修改），才能使得序列 $a$ 满足：任意相邻的元素都不是互质的。
> > 你的任务就是求出这个最小修改次数。
>
> >思路：这题思路很多，比如贪心或者dp都可（我就是dp水过去的，至今不解如何水过的）。我们就以贪心为例。
> >
> >我们考虑一个数时，如果修改它，是可以做到同时让他不和前一位和后一位互质。整体就是这个思路，然后就是卡人的点
> >
> >难点一 1是一定要修改的
> >
> >难点二 如何维护
> >
> >我们可以将数对作为考虑对象，更新其后一位,同时当有1时答案加一。我们要维护数对保证数对第一位不为1，这样我们在修改第二位的情况下，1和其它数就没区别了。
>
> code
>
> ```c++
> #include <bits/stdc++.h>
> using namespace std;
> using ll = long long;
> int n;
> int a[202020];
> int gcd(int a,int b)
> {
>     if(b==0)return a;
>     return gcd(b,a%b);
> }
> void Solve() {
>     cin >> n;
>     for (int i = 0; i < n; i++) {
>         cin >> a[i];
>     }
>     int res = 0;
>     for (int i = 0; i < n; i++) {
>         if (a[i] == 1) {
>             res++;
>             continue;
>         }
>         if (i + 1 < n && gcd(a[i], a[i + 1]) == 1) {
>             res++;
>             i++;//保证按数对处理
>         }
>     }
>     cout << res << '\n';
> }
> int main() {
>     ios::sync_with_stdio(false);
>     cin.tie(nullptr);
>     int T;
>     cin >> T;
>     while (T--) {
>         Solve();
>     }
> }
> ```
