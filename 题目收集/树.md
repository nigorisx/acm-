## p1:ifs优先搜索（树上dp）（模拟+dp）

有一棵由 n*+1 个顶点（*n* **为奇数**）组成的二叉树，其顶点的标签为 0,1,…,n0,1,…,*n*。树的每个顶点上**最多可以写一个字母**，所有顶点上最初都没有写任何东西。树的根是顶点 0。

在树中，顶点 00 是顶点 11 的父顶点，而所有其他顶点要么有 22 个子顶点，要么有 00 个子顶点。

鲍勃迷失在树的一个顶点上，他希望通过到达顶点 00 来逃离这棵树。对于大多数有常识的人来说，这非常容易。然而，由于鲍勃是个白痴，他创造了一种新的遍历树的方法，即引入 **“白痴优先搜索”**。

当鲍勃位于顶点 v*（*1≤v≤n1≤*v*≤*n*）时，鲍勃的移动步骤如下：

- 如果顶点 v 是叶子，那么鲍勃会移动到 *v* 的父顶点；
- 否则：
  - 如果没有在顶点 *v* 上写入任何内容，则鲍勃在顶点 *v* 上写入 "L"，并移动到 *v* 的左子顶点；
  - 如果 'L' 写在顶点 *v* 上，鲍勃将其改写为 'R'，并移动到 *v* 的右子顶点；
  - 如果 'R' 写在顶点 *v* 上，鲍勃将其擦除，并移动到 *v* 的父节点。

鲍勃移动到相邻顶点正好需要 11 秒，因此鲍勃执行 *x* 次移动正好需要 *x* 秒。

有研究表明，无论鲍勃从哪个顶点开始移动，鲍勃都能在有限的时间内（尽管可能大得令人费解）到达顶点 00。我们不知道是谁证明了这一点；当然不可能是鲍勃，但它确实被证明了。

对于每个顶点 *k*=1,2,…,*n*，请确定如果鲍勃从顶点 *k* 开始，到达顶点 00 所需的总时间，以秒为单位。由于数值可能很大，因此只要求计算取模 *$10^9$+7*。

```c++
#include <bits/stdc++.h>
using namespace std;
const int MOD = 1e9 + 7;
vector<int> dp; 
vector<int> ans;
vector<vector<int>> v;
void dfs1(int u) {
    if (v[u].empty()) {
        dp[u] = 1; 
        return;
    }
    int l = v[u][0], r = v[u][1];
    dfs1(l);
    dfs1(r);
    dp[u] = (dp[l] + dp[r] + 3) % MOD;
}//自下而上
void dfs2(int u, int pans) {
    ans[u] = (dp[u] + pans) % MOD;
    for (auto e : v[u]) {
        dfs2(e, ans[u]);
    }
}//自上向下
void solve() {
    int n;
    cin >> n;
    dp.assign(n + 1, 0);
    ans.assign(n + 1, 0);
    v.assign(n + 1, vector<int>());
    vector<int> isroot(n + 1, 0);
    for (int i = 1; i <= n; i++) {
        int l, r;
        cin >> l >> r;
        isroot[l] = 1;
        isroot[r] = 1;
        if (l != 0) {
            v[i].push_back(l);
        }
        if (r != 0) {
            v[i].push_back(r);
        }
    }
    int rt = 1;
    dfs1(rt);
    dfs2(rt, 0);
    for (int i = 1; i <= n; i++) {
        cout << ans[i] << " \n"[i == n];
    }
}
int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);
    
    int t;
    cin >> t;
    while (t--) {
        solve();
    }
    return 0;
}
```

***错误分析***

***这道题的情景是一种特殊的树上遍历，所以自然而然会想到dfs，又因为存在状态转移，所以可能会用dp。想到了要算每个点回到父节点所需的时间，但没在此时使用dp，dp的意识还是较弱.dp的思路明显比找规律好理解。除此之外，二叉树的保存可以单纯保存为l和r，不一定是固定模板的邻接表。***

**思考**

**树上的dp不一定是从下而上的，也有可能自上而下，要看一开始算出的状态在上方还是下方。比如此题先用自下而上算出每个节点到父节点所需时间，这是因为叶子节点到父节点的时间一定是1。再用自上而下算出答案，这是因为根节点的时间就是其到父节点的时间。**

## p2 Heapify 1（简易并查集）

给你一个长度为 n ∗ 的排列组合 a 。

您可以执行以下任意次数（可能为零）的操作：

- 选择索引 i （ 1≤i≤n2 ），然后交换 ai 和 a2i 。

例如，当 a=[1,4,2,3,5] 时，可以交换 a2 和 a4 ，使其成为 [1,3,2,4,5] ，但不能交换 a2 和 a3 。

请判断序列 a 能否按递增顺序排序。

∗ 长度为 n 的排列是由 1 至 n 之间的 n 个不同整数按任意顺序排列而成的数组。例如， [2,3,1,5,4] 是一个排列，但 [1,2,2] 不是一个排列（ 2 在数组中出现了两次）， [1,3,4] 也不是一个排列（ n=3 ，但数组中有 4 ）。

```c++
#include <bits/stdc++.h>
using namespace std;
int f(int x)
{
    while(x%2==0) x/=2;
    return x;
}
void solve()
{
    int n;
    cin>>n;
    vector<int>a(n+1);
    bool flag=true;
    for(int i=1;i<=n;i++)cin>>a[i];
    for(int i=1;i<=n;i++)
    {
        if(f(a[i])!=f(i))
        {
            flag=false;
            break;
        }
    }  
    if(flag)cout<<"YES"<<endl;
    else cout<<"NO"<<endl;  
}
int main()
{
    int t;
    cin>>t;
    while(t--)
    {
        solve();
    }
}
```

**记录这题的原因是因为这题没有第一时间看出是并查集。这题所有倍增索引下的元素都处于一个可以交换的集合中，所以只要考虑a[i]和i是否在同一集合中即可，所以想到用并查集中的find，因为将最前侧元素当作头节点，所以可以省去merge**

## p4 TreeScript（树上dp）

> [!note]
>
> ***题目描述***
>
> TreeScript is a programming language developed for maintaining tree structures. In this problem, we will learn how to create rooted trees in TreeScript. In TreeScript, all tree nodes are stored in memory. Each tree node has a number and the address of its parent node, and both are immutable, so they have to be determined when creating the node. In particular, the address of the root node’s parent node is empty. In order to access these nodes, the address of a node can be stored in a register. If there are m registers, the registers can be written as r[0], r[1], ..., r[m − 1]. Now let’s learn the node creation statement: $r[i] = create(r[j], k)$; where k is the node number, i and j are the indices of the registers, where $0 ≤ i, j < m$ and i = j is possible. The effect of this statement is that a node numbered k is created, whose parent address is stored in r[j], and then the new node’s address is stored in r[i]. Once each node has been created correctly, you do not need to store the address of any more nodes; they will automatically execute the pre-defined instructions. For reasons of space, we will learn about them later. To check your learning, you need to create a rooted tree of size n. At first, the system will automatically create the root node for you and store it in r[0]. So you only need to execute n − 1 additional creation instructions to create the tree. As you know, registers are very expensive, so you need to find the minimum value of the registers you need.
>
> ***思路***
>
> 画图可知，寄存器数目只有在分叉处才会有更新，而且需要多少寄存器和子树的形态有关系，所以考虑$树上dp$，一个点可能有多颗子树如何能保证寄存器数目最小化而又足够呢？第一想法，转移子树中最多的，验证此想法，由于子树的根节点不可删除在子树未遍历完时，所以会一直占着一个寄存器，所以当最大子树有1个寄存器，次大也有1个时，那么该树如果只有1个寄存器，无法保存根的信息，第一轮后根就丢失了，是不可以的，所以至少要2个寄存器。抽象出来，子树最大和次大需求量分别为$d_1$和 $d_2$,至少需要max($d_1,d_2+1$);究其根本，是因为当一个信息不只使用一次时，就不可在使用后立刻删除，此时若无多余寄存器，则需多取一个，显然有多子树的根节点就满足这种多次使用的情况。知道关系dfs遍历树然后自下而上更新dp数组即可，就很简单了

code

```c++
#include <bits/stdc++.h>
using namespace std;
vector<vector<int>>e;
vector<int>dp;
int  dfs(int u)
{
    if(e[u].size()==0)return dp[u]=1;
    vector<int>res;
    for(auto son:e[u])
    {    
        res.push_back(dfs(son));
    }
    sort(res.rbegin(),res.rend());
    dp[u]=res[0];
    if(res.size()>=2)
    dp[u]=max(dp[u],res[1]+1);
    return dp[u];
}
void solve()
{
    int n;
    cin>>n;
    e.assign(n + 1, vector<int>());
    dp.assign(n + 1, 0);
    int st;
    for(int i=1;i<=n;i++)
    {
        int to;
        cin>>to;
        e[to].push_back(i);
        if(to==0)st=i;
    }
    cout<<dfs(st)<<endl;
}
int main()
{
    int t;
    cin>>t;
    while(t--)
    {
        solve();
    }
}
```

## p5 [Tree Planting][#https://codeforces.com/problemset/problem/1187/E]（换根dp）

> [!note]
>
> >***题目描述***
> >
> >给你一棵树（一种无向连通且无环的图），它有 *n* 个顶点。你要在这棵树上玩一个游戏。
> >
> >游戏开始时，所有顶点都是白色的。第一回合，你选一个顶点涂成黑色。之后每回合，你必须选择一个与 **任意** 黑色顶点相邻（通过一条边相连）的白色顶点，将它涂成黑色。
> >
> >每次选顶点（包括第一回合），你都会获得一波积分，积分数等于包含该顶点的白色连通块的大小。游戏结束时，所有顶点都被涂成黑色。你的目标是让总积分尽可能高。
>
> >***思路***
> >
> >这道题的特点是当根节点固定，整个的总积分就是固定的，所以答案只跟根节点有关。那我们就会想到树状dp中的一种特殊类型，换根dp。利用两次dfs，将O($n^2$)的暴力变为O(n+n)的dp。这题还有个特殊的地方，节点相对根的深度就是其对总积分的贡献值。因此就和换根dp的例题一样了，套模板就行。
>
> >***code***
> >
> >```cpp

> >#define int long long
> >vector<vector<int>>e;
> >vector<int>siz;

> >int n;
